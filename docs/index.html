Overview

What is TSST?

TSST is a small language focused on readable syntax, simple types, packages, and a clean development experience. It runs through a Rust-built CLI and includes a VS Code extension for syntax coloring and autocomplete.

⌘

Clean Syntax

Typed declarations like cre_int, cre_str, and cre_dict.

↳

Functions

Use parameters, return values, and arrow return types like -> int.

▦

Data Structures

Arrays, dictionaries, indexing, mutation macros, and len().

⬡

Packages

Install GitHub packages with tsst install and import them with use "ui:ui";.

✦

Editor Support

VS Code language recognition, syntax highlighting, autocomplete, and snippets.

⚡

Fast CLI

Create projects with tsst init, install packages, and run files from any folder.

Install

Install for development

01

Clone the repo

git clone https://github.com/22aispro/TSST.git
cd TSST
02

Build the CLI

cargo build --release
03

Run a TSST file

.\target\release\tsst.exe path/to/main.tsst
Tip: Add target/release to your Windows PATH so you can run tsst main.tsst from any TSST project folder.

Projects

Create a TSST project

Create a project anywhere with tsst init.

Create an app

cd C:\Users\YourName\Desktop
tsst init MyApp
cd MyApp

Run it

tsst main.tsst

Generated structure

MyApp/
  main.tsst
  tsst.json
  packages/

Syntax

Variables

cre_int age = 37;
cre_str name = "John Doe";
cre_bool cool = true;

cre_arr nums = [1, 2, 3];

cre_dict user = {
    "name": "John Doe",
    "age": 37,
};

Syntax

Functions

fcn add (cre_int a, cre_int b) -> int {
    return a + b;
}

pub fcn main () {
    cre_int result = add(5, 10);
    cons!(result);
}

Syntax

If / Else

pub fcn main () {
    cre_int age = 37;

    if age > 18 {
        cons!("adult");
    } else {
        cons!("minor");
    }
}

Syntax

Loops

Classic for loop

for (cre_int i = 0; i < 5; i = i + 1) {
    cons!(i);
}

For-each loop

cre_arr nums = [10, 20, 30];

for (cre_int n in nums) {
    cons!(n);
}

Break and continue

for (cre_int i = 0; i < 10; i = i + 1) {
    if i == 3 {
        continue;
    }

    if i == 7 {
        break;
    }

    cons!(i);
}

Data

Arrays

pub fcn main () {
    cre_arr nums = [10, 20, 30];

    cons!(nums[0]);
    cons!(len(nums));

    push!(nums, 40);

    cons!(nums);
}

Data

Dictionaries

pub fcn main () {
    cre_dict user = {
        "name": "John Doe",
        "age": 37,
    };

    cons!(user["name"]);

    set!(user, "admin", true);

    for (cre_str key in user) {
        cons!(key);
        cons!(user[key]);
    }
}

Files

Imports

Relative imports load another .tsst file from your project. 

use "math.tsst";

pub fcn main () {
    cre_int x = add(5, 10);
    cons!("x: " + x);
}

Package imports

Package imports load files from the packages/ folder. 

use "ui:ui";

pub fcn main () {
    ui_success("Loaded package");
}

Built-ins

Built-ins and macros

cons!(value)

Prints a value to the console.

len(value)

Returns the length of a string, array, or dictionary.

push!(array, value)

Adds a value to an array.

set!(dict, key, value)

Sets a dictionary key to a value.

Packages

Install packages from GitHub

TSST uses a tsst.json file to install dependencies. Packages are cloned into your local packages/ folder.

tsst.json

{
  "name": "MyApp",
  "version": "0.1.0",
  "dependencies": {
    "ui": "github:22aispro/Quartus.ui"
  }
}

Install dependencies:

tsst install

After install, the package is placed here:

MyApp/
  packages/
    ui/
      ui.tsst

Use the package in your TSST file:

use "ui:ui";

pub fcn main () {
    ui_success("Package loaded");
}

Official Package

Quartus.ui

Quartus.ui is a console UI package for TSST. It gives you titles, cards, labels, progress bars, checkboxes, and status messages.

Install

{
  "name": "MyApp",
  "version": "0.1.0",
  "dependencies": {
    "ui": "github:22aispro/Quartus.ui"
  }
}

Command

tsst install

Example

use "ui:ui";

pub fcn main () {
    ui_app_start("Quartus UI Demo");

    ui_title("Dashboard");

    ui_card("Profile", "John Doe");
    ui_number("Age", 37);
    ui_bool("Admin", true);

    ui_success("Quartus.ui loaded from GitHub");

    ui_app_end();
}

Common functions

ui_app_start(app_name)

Prints an app banner and start message.

ui_title(text)

Prints a styled title section.

ui_card(title, body)

Prints a simple console card.

ui_success(text)

Prints a success message.

ui_number(label, value)

Prints a label with an integer value.

ui_app_end()

Prints an ending message and divider.

Editor

VS Code extension

TSST includes a VS Code extension for syntax coloring, snippets, autocomplete, and .tsst file recognition.

Development mode

cd tools/vscode-tsst
code .

Press F5 in VS Code to open the Extension Development Host.

Install permanently with VSIX

cd tools/vscode-tsst
vsce package
code --install-extension .\vscode-tsst-0.0.1.vsix
