Overview

What is TSST?

TSST is a small language focused on readable syntax, simple types, package-style imports, and a clean development flow. It runs through a Rust-built CLI and can either interpret TSST files or compile them into executable apps.

⌘

Aesthetic Syntax

Typed declarations like cre_int, cre_str, cre_bool, cre_arr, and cre_dict.

↳

Functions

Use parameters, return values, and arrow return types like -> int.

▦

Data Structures

Arrays, dictionaries, indexing, mutation macros, and len().

⚙

Compiled Builds

tsst build compiles TSST into Rust and creates a release executable.

◇

Lightweight

TSST keeps the syntax small, readable, and easy to experiment with.

✦

Editor Support

VS Code language recognition, syntax highlighting, autocomplete, and snippets.

Install

Install for development

01

Clone the repo

git clone https://github.com/22aispro/TSST.git
cd TSST
02

Build the CLI

cargo build --release
03

Run TSST

.\target\release\tsst.exe path/to/main.tsst
Tip: Add target/release to your Windows PATH so you can run tsst from any TSST project folder.

Run & Build

Run TSST files or build executables

TSST currently has two execution paths: interpreted mode and compiled mode.

Interpreter mode

tsst run main.tsst

This runs TSST directly through the interpreter. Console programs work here.

Compiled mode

tsst build main.tsst
.\build\release\main.exe

This compiles TSST into Rust and builds a release executable. GUI apps currently require compiled mode.

Build output

MyApp/
  build/
    rust/
      Cargo.toml
      src/
        main.rs
    release/
      main.exe

Syntax

Variables

pub fcn main () {
    cre_int age = 37;
    cre_str name = "John Doe";
    cre_bool cool = true;

    cre_arr nums = [1, 2, 3];

    cre_dict user = {
        "name": "John Doe",
        "age": 37,
    };

    cons!(name);
}

Syntax

Functions

fcn add (cre_int a, cre_int b) -> int {
    return a + b;
}

pub fcn main () {
    cre_int result = add(5, 10);
    cons!(result);
}

Input

Read user input

TSST supports interactive terminal input using built-in functions.

input_str(prompt)

Prints a prompt and returns the user's input as a string.

input_int(prompt)

Prints a prompt and returns the user's input as an integer.

Example

pub fcn main () {
    cre_str name = input_str("Name: ");
    cre_int age = input_int("Age: ");

    cons!("Hello " + name);
    cons!("Age: " + age);
}

Syntax

If / Else

pub fcn main () {
    cre_int age = 37;

    if age > 18 {
        cons!("adult");
    } else {
        cons!("minor");
    }
}

Syntax

Loops

Classic for loop

pub fcn main () {
    for cre_int i = 0; i < 5; i = i + 1 {
        cons!(i);
    }
}

For-each loop

pub fcn main () {
    cre_arr nums = [10, 20, 30];

    for cre_int n in nums {
        cons!(n);
    }
}

While loop

pub fcn main () {
    cre_int i = 0;

    while i < 5 {
        cons!(i);
        i = i + 1;
    }
}

Break and continue

pub fcn main () {
    for cre_int i = 0; i < 10; i = i + 1 {
        if i == 3 {
            continue;
        }

        if i == 7 {
            break;
        }

        cons!(i);
    }
}

Data

Arrays

pub fcn main () {
    cre_arr nums = [10, 20, 30];

    cons!(nums[0]);
    cons!(len(nums));

    push!(nums, 40);

    cons!(nums);
}

Data

Dictionaries

pub fcn main () {
    cre_dict user = {
        "name": "John Doe",
        "age": 37,
    };

    cons!(user["name"]);

    set!(user, "admin", true);

    for cre_str key in user {
        cons!(key);
    }
}

Files

Imports

Relative imports load another .tsst file from your project.

Package imports

Package imports load files from the packages/ folder.

GUI

GUI apps

TSST can build simple GUI apps in compiled mode. GUI apps use generic runtime functions, while app logic, layout, and style are written in TSST.

Important: GUI apps currently require tsst build. Use tsst run for console programs and tsst build for GUI programs.

Minimal GUI app

pub fcn main () {
    gui_window("My App", 600, 500);

    gui_title("Hello GUI");
    gui_text("This is a TSST GUI app.");

    gui_show();
}

Packages

Install packages from GitHub

TSST uses a tsst.json file to install dependencies. Packages are cloned into your local packages/ folder.

tsst.json

{
  "name": "MyApp",
  "version": "0.1.0",
  "dependencies": {
    "quartus": "github:22aispro/Quartus"
  }
}

Commands

tsst install

Installs dependencies from tsst.json.

tsst list

Lists installed packages in the packages/ folder.

tsst remove <package>

Removes an installed package.

tsst update

Deletes installed packages and reinstalls everything from tsst.json.

Architecture

How TSST builds apps

TSST separates compiler logic from app logic. The compiler turns TSST into Rust calls. The generated runtime provides generic language and GUI features. Your TSST file controls the actual app behavior and style. 

compiler.rs
  compiles TSST AST into Rust

generated_runtime.rs
  provides generic runtime and GUI engine

main.tsst
  controls app logic, layout, and visual style
Goal: App-specific behavior should be written in TSST, not hardcoded into the Rust compiler.

Roadmap

What’s next?

Move the generated runtime string into a real reusable Rust runtime crate.

Add GUI callbacks so TSST functions can run when buttons are clicked.

Add generic file and folder APIs for workspace apps.

Add generic audio APIs without per-app Rust changes.

Improve package commands like search and publish.

Smarter VS Code autocomplete for installed package functions and GUI APIs.

Create official releases for the TSST CLI.

TSST is experimental, built for learning, language design, and fun.
